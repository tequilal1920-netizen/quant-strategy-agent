# 数据看板

## 目标

维护统一、可审计的数据入口，为下游研究模型提供带日期、来源和质量状态的事实层。

## 输入

- 目标页面、研究截止日、频率、市场和数据口径。
- 本地数据库、正式 JSON 快照及允许访问的外部数据源。
- 可选的行业、资产、证券或新闻筛选条件。

## 输出

- 宏观、全球市场、行业、大宗商品、个股和新闻事件的结构化快照与可视化。
- AI 监控外部实时页面及其可用性状态。
- `as_of`、`generated_at`、覆盖率、来源、单位、频率和质量说明。
- 可供主页及其他模型消费的稳定字段契约。

## 工作流

1. 阅读 `references/module-map.md`、`model/data_dashboard/README.md` 和当前状态文件。
2. 确认目标二级页面及其数据契约；不要用邻近字段或旧页面名称推测映射。
3. 检查来源、截止日、频率、单位、缺失率和滞后；缺失值必须显式呈现。
4. 只修改正式抓取、清洗或快照入口，保持页面字段与下游消费者兼容。
5. 重新生成受影响快照，并验证结论卡、图表、状态点和缓存响应。
6. AI 监控只嵌入配置的公网页面；失败时展示红色状态和可独立打开的链接。
7. 运行发布校验与统一导航测试；涉及 UI 时遍历全部七个二级页面。

## 约束

- 不把缺失值、旧缓存或失败响应伪装成最新数据。
- 行业深度报告必须经过 `data_quality.py`。
- 状态颜色固定为绿色正常、蓝色运行、红色异常。
- 中文字体为楷体，英文和数字为 Arial；统一卡片、最小字号与图表样式。
- 数据库存放在 `database/`；不要复制、压缩或删除大型数据库。
