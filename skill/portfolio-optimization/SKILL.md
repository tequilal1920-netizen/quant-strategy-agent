# 组合优化

## 目标

把上游预期收益、风险与约束转化为可复现的候选组合、目标权重和配置策略。

## 输入

- 研究截止日、资产池、预期收益、协方差或风险模型。
- 基准、当前持仓、风险预算、权重边界、换手和交易成本。
- 求解器、目标函数、候选数量和压力情景。

## 输出

- 优化求解状态、可行性、候选方案和约束残差。
- 选定配置策略的目标权重、风险贡献、换手、成本和回测摘要。
- 供主页展示的日度、周度、月度最终组合。

## 工作流

1. 阅读 `references/module-map.md`、模型 README 和当前状态文件。
2. 固定输入时间点、资产集合、基准和成本口径；禁止用测试集选参。
3. 验证预期收益和风险矩阵的资产顺序、单位、缺失值与正定性处理。
4. 运行正式优化引擎，保留所有候选方案、求解状态和约束残差。
5. 检查权重和、上下界、风险预算、换手、成本敏感性和压力情景。
6. 将资产池、风险约束、求解、权重和回测图表保留在两个新页面的页内控件中。
7. 验证缓存键、重复点击、候选切换、页面跳转和最终主页联动。

## 约束

- 不得隐瞒不可行、未收敛或约束残差超限的求解结果。
- 测试集只做最终报告，不用于挑选求解参数或候选方案。
- 状态颜色固定为绿色正常、蓝色运行、红色异常。
- 中文楷体、英文和数字 Arial；遵守统一看板布局。
