# Audit Standard

本文件定义所有策略 notebook 和模型实现必须接受的审计。

## P0 致命问题

发现任一 P0，必须停止进入模型效果迭代：

- 使用未来收益构造特征。
- 使用全样本均值/标准差做历史标准化。
- 财务数据未按公告日滞后。
- 训练集和测试集泄漏。
- 调仓日使用不可交易价格。
- 明文写入账号、密码、token。
- notebook 运行失败但被标记成功。

## P1 严重问题

必须修复或由用户明确接受风险：

- 幸存者偏差。
- 指数成分使用最新成分回看历史。
- 未计交易成本。
- 停牌、涨跌停处理缺失。
- 缺失值处理导致样本隐形筛选。
- 付费 API 未缓存。
- 关键参数无证据且未进入待审批假设。

## P2 中等问题

可以进入下一阶段，但必须记录：

- 年化口径不统一。
- 图表标签不清晰。
- 输出文件命名不规范。
- 部分诊断缺失。
- 样本期过短。

## P3 低风险问题

主要影响可读性：

- 文档格式不统一。
- 变量命名不够清楚。
- 结果摘要不够简洁。

## 审计输出要求

每个问题必须包含：

- 问题编号。
- 严重等级。
- 发生位置。
- 证据。
- 影响。
- 修复方案。
- 是否需要用户批准。
